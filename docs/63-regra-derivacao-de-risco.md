# Regra: Derivacao de Risco

Sempre que possivel, o sistema deve derivar indicadores de risco.

Exemplos:

- crianca pode ser derivado de data_nascimento
- idoso pode ser derivado de data_nascimento

Diretriz:

- preferir calculo sobre armazenamento
- preferir classificacao em tempo de consulta
- evitar rotulos sensiveis permanentes sem necessidade real

Objetivo:

- reduzir dados sensiveis persistidos
- manter a analise util
- melhorar aderencia a LGPD
