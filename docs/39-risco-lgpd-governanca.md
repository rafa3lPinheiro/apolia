# Risco: LGPD e Governanca

Objetivo:

- registrar os limites de uso da analise de risco
- deixar claro que a leitura operacional depende de governanca

## Diretrizes iniciais

- risco deve existir para apoiar a operacao, nao para automatizar exclusao
- toda coleta de dado sensivel precisa ter finalidade clara
- acesso a dado sensivel precisa ser mais restrito que acesso operacional comum
- exibicao em tela deve ser minima
- auditoria de consulta e alteracao deve ser considerada requisito

## Perguntas que esta nota precisa fechar

- quais perfis podem ver cada tipo de dado sensivel
- quais indicadores podem ser exibidos sem expor dado individual
- quais dados entram no MVP
- quais campos exigem protecao reforcada desde a primeira versao
